### 1. Data Collection
For the given date and city (using exact latitude, longitude, and timezone), the system first uses the high-precision **Swiss Ephemeris** to fetch:
*   **Sunrise** and **Sunset** for the current day.
*   **Next Sunrise** for the following day.

### 2. Identifying Transition Points
Instead of dividing the day into fixed intervals, the system looks at the exact timestamps where any of the following astrological elements change between the current Sunrise and the Next Sunrise:
*   **Tithi changes** (Lunar day)
*   **Nakshatra changes** (Lunar mansion)
*   **Lagna changes** (Ascendant sign)

It takes the Sunrise time, the Next Sunrise time, and all these transition times, and sorts them chronologically. These become the boundaries for our time intervals.

### 3. The Core "Panchaka Rahita" Calculation
For every interval created by these transition points, the system checks the midpoint of the interval. At that exact midpoint, it calculates four astrological indices:

1.  **Tithi Index (`tIndex`)**: Values from 1 to 30 (1 = Pratipada, 15 = Purnima, 30 = Amavasya).
2.  **Vara Index (`vIndex`)**: The day of the week (1 = Sunday, 2 = Monday, ..., 7 = Saturday).
3.  **Nakshatra Index (`nIndex`)**: Values from 1 to 27 (1 = Ashwini, ..., 27 = Revati).
4.  **Lagna Index (`lIndex`)**: Values from 1 to 12 (1 = Aries/Mesha, ..., 12 = Pisces/Meena).

### 4. The Sum and Modulo Logic
The system sums these four values together:
**`Sum` = `tIndex` + `vIndex` + `nIndex` + `lIndex`**

Then, it divides the sum by 9 and looks at the **remainder**:
**`Remainder` = `Sum` % 9**

### 5. Final Categorization (The Result)
Based on the exact remainder, the time interval is categorized into one of the following Muhurats:

*   **Remainder `1`**: Mrityu (Danger)
*   **Remainder `2`**: Agni (Risk)
*   **Remainder `4`**: Raja (Bad)
*   **Remainder `6`**: Chora (Evil)
*   **Remainder `8`**: Roga (Disease)
*   **Remainders `0, 3, 5, 7`**: Shubha (Good)
